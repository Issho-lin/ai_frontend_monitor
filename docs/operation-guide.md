## 1. 产品概述

AI 前端性能监控平台是一套**面向前端团队**的端到端性能治理系统。它不只是"看图"，而是打通从数据采集 → 异常发现 → Agent 诊断 → 治理任务 → 修复验证 → 复盘沉淀的全链路。

### 核心能力

| 能力 | 说明 |
|------|------|
| **Web 性能采集** | 自动采集 LCP / INP / CLS / FCP / TTFB 等 Web Vitals |
| **错误监控** | JS Error、Promise 异常、资源加载失败，自动归类分组 |
| **接口分析** | API P50 / P95 耗时、错误率、超时率、Trace 覆盖率 |
| **AI 诊断** | Agent 基于证据生成结构化诊断报告，包含根因候选和置信度 |
| **告警** | 固定阈值、环比/同比突增，支持收敛、静默、多渠道通知 |
| **治理任务** | 创建任务 → 分配负责人 → 修复验证 → 复盘沉淀的闭环管理 |
| **版本对比** | 发布前后性能差异对比，精准判断是否由新版本引入 |

### 目标用户

| 角色 | 如何使用 |
|------|----------|
| **前端研发** | 查看页面/接口/错误详情，处理治理任务 |
| **前端负责人** | 关注总览健康度、告警、治理任务优先级 |
| **SRE / DevOps** | 监控发布后性能变化，配置告警和通知渠道 |

## 4. 监控总览

**路径**：首页 → 监控总览

总览页在 30 秒内回答一个问题：**当前项目是否健康？**

### 页面结构

#### ① 健康状态条

横向展示 4 个关键信号：

| 信号 | 说明 |
|------|------|
| **项目健康度** | 圆点颜色指示 — 🟢 健康 / 🟡 需要关注 / 🔴 严重 |
| **当前活跃告警** | 未解决告警数量 |
| **受影响用户** | 命中性能劣化条件的去重用户数 |
| **最近一次发布** | 最近发布的版本号和发布时间 |

#### ② 核心指标卡

6 张指标卡，展示 P75 核心性能指标：

- **LCP P75** — 最大内容绘制，Good ≤ 2.5s
- **INP P75** — 交互响应，Good ≤ 200ms
- **CLS P75** — 布局稳定性，Good ≤ 0.1
- **JS Error Rate** — 错误会话率，Good ≤ 0.1%
- **API P95** — 接口耗时，由业务配置
- **资源失败率** — 静态资源加载失败，Good ≤ 0.2%

每张卡显示**当前值 + 环比变化**（↑ 变差 / ↓ 改善）。

#### ③ 核心指标趋势图

24 小时折线图，同时展示 LCP / INP / API P95 / JS Error Rate 四条曲线。可切换查看不同指标的走势。

#### ④ Agent 摘要

AI 自动总结当前最值得关注的问题，包括：

- **结论** — 一句话说明什么问题
- **详情** — 主要证据简述
- **置信度** — 高 / 中 / 低
- **更新时间** — 诊断报告生成时间

点击"查看告警"跳转到告警详情；"分析页面"跳转到页面性能列表。

#### ⑤ TOP 问题

按影响面排序的前 5 个问题，包含页面、接口、资源三类，显示当前指标值。

## 5. 页面性能分析

**路径**：页面性能 → 点击进入任意页面查看详情

### 5.1 页面列表

展示所有被监控页面的性能汇总：

| 列 | 说明 |
|----|------|
| 页面路由 | 归一化后的路由，如 `/home`、`/product/:id` |
| PV | 页面浏览量 |
| 受影响用户 | 命中不达标条件的去重用户数 |
| LCP / INP / CLS P75 | 各指标的 P75 值，LCP 有颜色标签 |
| JS Error | 错误率 |
| API P95 | 接口 P95 耗时 |
| 达标率 | LCP/INP/CLS 均处于 Good 的访问占比 |
| 趋势 | 迷你趋势图 + 升降箭头 |

**交互**：
- 点击行或右侧箭头 → 进入页面详情页
- 顶部搜索框可按路由名称过滤
- 指标卡支持切换趋势图显示的指标

### 5.2 页面详情

**路径**：`/pages/:route`

页面详情是排查"某个页面为什么慢"的核心工作台，采用**左右分栏**布局：

#### 左侧 — 指标与诊断

1. **6 项指标卡** — 当前页面的 LCP / INP / CLS / JS Error / API P95 / 达标率
2. **Web Vitals 趋势** — 24 小时折线图，含"Good"基准线
3. **设备分布** — 饼图，移动端 / 桌面端 / 平板占比
4. **地区分布** — 柱状图，各区域流量占比
5. **首屏资源瀑布** — 按加载耗时排序的关键资源清单，展示体积、耗时、缓存命中率、失败率
6. **关联接口** — 该页面调用的接口 P50/P95 及错误率

#### 右侧 — Agent 诊断

自动拉取与当前页面最相关的告警诊断报告，包含：

| 模块 | 说明 |
|------|------|
| 结论 | 一句话结论，如"移动端 /home LCP 在 v1.18.0 后劣化" |
| 影响面 | 影响页面、用户、会话、版本 |
| 证据 | 编号证据列表（e1/e2/e3…），每条可追溯到原始查询 |
| 根因候选 | 按置信度排序的推测原因 |
| 建议动作 | 可操作的具体修复建议 |
| 缺失证据 | 无法获取的数据（如 sourcemap 缺失） |
| 工具调用轨迹 | 可折叠的审计面板，展示 Agent 调用了哪些只读工具 |

## 6. 错误分析

**路径**：错误分析

### 功能说明

按错误分组聚合 JS 异常、Promise 拒绝和资源加载失败，帮助快速定位：

- 哪个错误出现最多
- 影响了多少用户
- 出现在哪个页面和版本
- 错误是否已处理

### 页面内容

1. **错误趋势图** — 24 小时堆叠柱状图，按 JS Error / Promise / Resource 分类
2. **错误分组列表**

| 列 | 说明 |
|----|------|
| 错误信息 | 完整错误 message，支持复制 |
| 类型 | JS Error / Promise / 资源错误 |
| 页面 | 错误发生的页面路由 |
| 版本 | 引入该错误的发布版本 |
| 次数 | 错误触发总次数 |
| 受影响用户 | 去重用户数 |
| 状态 | 未处理 / 观察中 / 已处理 |
| 操作 | 查看堆栈 / 创建任务 |

### 筛选

- **搜索** — 按错误信息关键词搜索
- **类型过滤** — 全部 / JS / Promise / 资源

## 7. 接口分析

**路径**：接口分析

### 功能说明

按归一化后的 API 路由聚合接口性能数据，帮助发现：

- 哪些接口 P95 最慢
- 哪些接口错误率/超时率偏高
- 哪些接口 Trace 覆盖率不足

### 页面内容

1. **P50 / P95 对比柱状图** — 各接口 P50 与 P95 对比
2. **重点接口趋势** — 示例展示 P95 的 24 小时走势
3. **接口明细表**

| 列 | 说明 |
|----|------|
| 接口 | 归一化 API 路由 |
| Method | GET / POST 等 |
| P50 / P95 | 中位数与 P95 耗时（ms） |
| 错误率 | 百分比，颜色区分严重程度 |
| 超时率 | 超过超时阈值的请求占比 |
| QPS | 每秒请求数 |
| Trace 覆盖率 | 带追踪 ID 的请求占比 |

### 筛选

- **搜索** — 按接口路径搜索

## 8. 版本对比

**路径**：版本对比 → 点击进入版本详情

### 8.1 版本列表

展示所有发布记录：

| 列 | 说明 |
|----|------|
| 版本 | 版本号，如 `v1.18.0` |
| 环境 | 生产 / 预发 |
| 发布时间 | 精确到分钟 |
| 发布人 | 发布系统 / 发布人姓名 |
| Commit | Git commit 短哈希 |
| 灰度 | 当前灰度比例 |
| 状态 | 已发布 / 灰度中 / 已回滚 |
| 活跃告警 | 该版本引发的未解决告警数 |
| 性能变化 | 性能升降趋势（红升/绿降） |

**交互**：
- 点击行 → 进入版本对比详情
- 搜索可按版本号或 commit 搜索
- 环境筛选器：全部 / 生产 / 预发

### 8.2 版本对比详情

**路径**：`/releases/:version`

选择本版本后，默认与**上一版本**对比，展示：

1. **六项指标对比** — LCP / INP / CLS / API P95 / JS Error Rate / 页面达标率
   - 显示"发布前 24h 基线"vs"发布后至今"的 P75 差异
   - 显著劣化项标红

2. **页面差异** — 按路由列出各页面指标变化幅度

3. **资源 Diff** — 新增 / 删除 / 体积变化的资源清单，标注体积、hash、chunk 归属

4. **错误 Diff** — 新出现 / 消失 / 次数变化的错误分组

5. **接口 Diff** — 新增接口 / 耗时明显变化的接口

6. **关联告警** — 两次发布之间触发的所有告警清单，可跳转告警详情

**下钻链路**：从任一 diff 行点击进入 Pages / Errors / APIs 详情，时间自动锁定到该版本观察窗口。

## 9. 告警中心

**路径**：告警中心

### 9.1 告警列表

展示所有告警，支持三级筛选：

| 筛选器 | 说明 |
|--------|------|
| 关键词搜索 | 按告警标题搜索 |
| 严重级别 | S0（严重）/ S1（高）/ S2（中）/ S3（低） |
| 状态 | 触发中 / 已认领 / 已恢复 / 已静默 |

告警列表关键列：

| 列 | 说明 |
|----|------|
| 级别 | S0-S3 标签 |
| 告警 | 标题 + ID + 首次触发时间 |
| 页面 | 影响的路由 |
| 指标 | 触发告警的指标名称 |
| 当前值 / 基线 | 红色突出当前值，灰色显示基线 |
| 影响用户 | 去重用户数 |

| 状态 | 触发中 / 已认领 / 已恢复 / 已静默 |
| 操作 | 诊断按钮 → 进入告警详情 |

### 9.2 告警详情

**路径**：`/alerts/:id`

告警详情是处理性能异常的核心工作台，包含以下模块：

#### 顶部 — 告警摘要

- 告警 ID、严重级别、状态标签
- 标题、页面、指标、版本、首次触发时间
- **操作按钮**：创建治理任务 / 认领 / 静默 1 小时 / 标记误报

#### 指标数据

4 张指标卡：

| 指标卡 | 内容 |
|--------|------|
| 当前值 | 触发告警时的指标值 |
| 基线 | 过去 7 天 P75 |
| 变化幅度 | 与基线的差异百分比及持续时长 |
| 影响用户 | 去重用户数 + 受影响会话占比 |

#### 指标趋势图

24 小时折线图，展示触发告警指标的走势，含阈值参考线。

#### 告警收敛

展示本告警聚合了多少条子告警，来源哪些规则，可展开查看每条子告警的分群信息（如 iOS / Safari、Android / Chrome 等）。

#### 时间线

按时间顺序展示关键事件：

| 事件类型 | 图标颜色 | 说明 |
|----------|----------|------|
| 发布 | 蓝色 | 版本发布时间 |
| 异常开始 | 橙色 | 指标首次超阈值 |
| 告警触发 | 红色 | 告警事件生成 |
| Agent 诊断 | 绿色 | 诊断开始 / 报告完成 |
| 任务创建 | 灰色 | 治理任务创建 |

#### Agent 诊断报告

与页面详情右侧相同，展示完整结构化报告（结论、影响面、证据、根因候选、建议动作、缺失证据）。

#### Agent 工具调用轨迹

可折叠面板，展示 Agent 在诊断过程中调用的只读工具：

| 列 | 说明 |
|----|------|
| # | 调用序号（t1/t2…） |
| 工具 | 只读工具名称，如 `query_metrics`、`get_release_diff` |
| 入参 | 查询条件摘要 |
| 耗时 | 查询耗时（ms） |
| 输出摘要 | 关键结果概要 |

#### 协作区

- 评论输入框 — 添加处理记录或讨论
- 评论列表 — 按时间倒序展示所有协作记录

## 10. 治理任务

**路径**：治理任务

### 10.1 任务列表

展示所有治理任务，支持搜索和按状态筛选：

| 列 | 说明 |
|----|------|
| ID | 任务唯一编号 |
| 任务 | 标题 + 页面 + 来源告警（可跳转）+ 创建时间 |
| 负责人 | 当前认领人 |
| 优先级 | P0（最高）/ P1 / P2 / P3 |
| 验收指标 | 修复目标和观察窗口 |
| 截止时间 | 可选 |
| 状态 | 待认领 / 修复中 / 验证中 / 已关闭 / 再次触发 |

**操作**：
- 点击行 → 进入任务详情
- 点击"新建任务" → 手动创建
- 来源告警链接 → 跳回告警详情

### 10.2 任务状态机

```
  ┌──────────┐
  │  Open     │  已生成任务，等待认领
  └────┬─────┘
       │ 指派负责人
       ▼
  ┌──────────┐
  │InProgress│  修复中，可关联 commit
  └────┬─────┘
       │ 提交修复版本
       ▼
  ┌──────────┐
  │Verifying │  进入观察窗口
  └────┬─────┘
 达标 │       │ 不达标
     ▼       ▼
 ┌──────────┐ ┌──────────┐
 │ Resolved │ │ Reopened │──► 回到 InProgress
 └──────────┘ └──────────┘
```

### 10.3 任务详情

**路径**：`/tasks/:id`

| 模块 | 说明 |
|------|------|
| 问题摘要 | Agent 诊断结论引用 |
| 影响面 | 影响用户数、会话、页面 |
| 证据链 | 可点击回溯到原始查询的证据 ID 列表 |
| 验收指标 | 修复目标 + 观察窗口 + 当前值对比图 |
| 修复记录 | 处理动作、关联 commit、发布时间 |
| Agent 诊断 | 引用来源告警的诊断报告（可点击跳转） |
| 关联告警 | 来源告警信息 |

### 10.4 创建治理任务

**从告警创建**（推荐）：
1. 在告警详情点击"创建治理任务"
2. 弹出预填表单（字段来自 Agent 报告）：
   - **任务标题** — 从 Agent 结论自动生成，可修改
   - **负责人** — 从项目成员中选择
   - **优先级** — 默认跟随告警级别（S0/S1 → P0，S2 → P1）
   - **验收指标** — 从 Agent 建议动作中抽取
   - **截止时间** — 可选
   - **关联证据** — 自动带入证据 ID 列表
3. 修改后点击"创建任务"

**手动创建**：
- 在治理任务列表点击右上角"新建任务"

## 11. 项目设置

**路径**：项目设置

通过 Tab 组织，分为 7 个子面板：

### 11.1 采样配置

控制 SDK 采集的采样率，按事件类型分别设置：

| 事件类型 | 说明 |
|----------|------|
| Web Vitals | 性能指标采集，默认 100% |
| PV / 路由 | 页面浏览事件，默认 100% |
| API 事件 | 接口性能数据，默认 50% |
| 错误事件 | 错误采集，默认 100% |
| 资源事件 | 资源加载数据，默认 20% |

顶部显示实时上报统计（QPS、已接收、丢弃、限流命中、Beacon/Fetch 比例）。

### 11.2 路由归一化

将高基数动态路由归并为模板。例如：

| 匹配正则 | 归一化为 | 示例 |
|----------|----------|------|
| `/product/\d+` | `/product/:id` | `/product/8823` |
| `/user/\d+/orders` | `/user/:uid/orders` | `/user/42/orders` |

每条规则可独立开关。归一化后，页面详情按模板路由聚合。

### 11.3 脱敏规则

配置上报数据的脱敏策略，支持按字段和正则模式设置：

| 字段 | 示例模式 | 说明 |
|------|----------|------|
| URL 参数 | `token\|password\|access_token` | 脱敏 URL 中的敏感参数 |
| Header | `Authorization\|Cookie` | 脱敏请求头 |
| 错误堆栈 | `/tmp/.*\|/home/.*` | 脱敏敏感路径 |

### 11.4 告警规则

管理所有告警触发规则：

| 字段 | 说明 |
|------|------|
| 规则名 | 人类可读名称 |
| 指标 | LCP P75 / API P95 / JS Error Rate 等 |
| 触发方式 | 固定阈值 / 环比变化 / 同比变化 |
| 条件 | 阈值或变化幅度 |
| 持续时间 | 满足条件持续多久后触发 |
| 严重级别 | S0 / S1 / S2 / S3 |
| 通知渠道 | 从已配置渠道中多选 |
| 启用状态 | 开 / 关 |

**新建/编辑规则**：点击右侧按钮打开抽屉表单，填写所有字段后保存。

### 11.5 通知渠道

配置告警通知的目标渠道：

| 渠道类型 | 示例 |
|----------|------|

每个渠道可独立开关、测试、编辑。

### 11.6 发布数据

## 12. 接入验证

**路径**：`/onboarding` 或 项目设置 → 接入验证

接入验证是 9 步检查清单，分为两组：

### 第一组 — SDK 上报（前 5 步）

| 步骤 | 检查项 | 状态说明 |
|------|--------|----------|
| Step 1 | **Project Token** | `Passed` — 事件已使用 Token 签名 |
| Step 2 | **域名 / CORS** | `Passed` — 业务域名已在白名单 |
| Step 3 | **Web Vitals** | `Receiving` — 正在收到 LCP / INP / CLS 数据 |
| Step 4 | **JS Error** | `Waiting` — 30 分钟内未收到错误事件 |
| Step 5 | **API Timing** | `Receiving` — 正在收到接口耗时数据 |

### 第二组 — 发布数据（后 4 步）

| 步骤 | 检查项 | 状态说明 |
|------|--------|----------|
| Step 6 | **Release 元数据** | `Passed` — 版本号、Commit 已收录 |
| Step 7 | **构建产物 manifest** | `Passed` — 资源清单已收到 |
| Step 8 | **基础资源 diff** | `Passed` — 相对上一版本的变化已计算 |
| Step 9 | **Sourcemap** | `Warning` — 部分 sourcemap 缺失 |

### 状态颜色

| 状态 | 颜色 | 含义 |
|------|------|------|
| `Passed` | 绿色 | 通过 |
| `Receiving` | 蓝色 | 正在接收数据 |
| `Waiting` | 灰色 | 尚未收到数据 |
| `Warning` | 黄色 | 部分通过，有缺失 |
| `Failed` | 红色 | 不通过 |

### SDK 接入示例

页面底部提供接入代码片段：

```javascript
// npm 安装

// 初始化
import { init } from '@ai-frontend-monitor/sdk'

init({
  projectToken: 'PROJECT_TOKEN',
  env: 'production',
  release: 'v1.18.0',
  sampleRate: 1.0,
  reportUrl: 'https://ingest.example.com/report'
})
```

## 13. 核心工作流：从异常到修复

以下是日常使用的典型场景流程：

### 场景 1：发布后核心页面变慢

1. **查看总览** — 健康状态条显示"需要关注"，Agent 摘要提示"移动端 /home LCP 在 v1.18.0 发布后劣化"
2. **分析页面** — 进入页面性能列表，点击 `/home` 查看详情，查看资源瀑布发现 banner.png 从 180KB 增至 1.9MB
3. **查看告警** — 进入告警中心，点击"移动端 /home LCP P75 劣化"查看告警详情
4. **查看诊断** — 右侧 Agent 诊断报告显示根因候选"首屏图片资源体积增大"，置信度 high
5. **创建治理任务** — 点击"创建治理任务"，确认负责人和验收指标
6. **修复发布** — 修复后发布新版本，在告警详情关联修复版本
7. **验证修复** — 任务进入验证状态，平台按观察窗口对比修复前后指标
8. **关闭任务** — 指标达标后关闭任务，填写复盘

### 场景 2：排查线上错误

1. **错误分析** — 进入错误分析页面，查看错误趋势图
2. **定位错误** — 按错误信息搜索，找到高发的错误分组
3. **查看详情** — 查看错误堆栈、影响用户、所在页面和版本
4. **创建任务** — 点击"创建任务"将其加入治理任务

### 场景 3：版本发布前自查

1. **版本对比** — 进入版本对比列表，选择新版本
2. **查看指标对比** — 检查 LCP / INP / API P95 是否有劣化
3. **查看资源 diff** — 确认新增/删除资源是否符合预期
4. **查看错误 diff** — 确认是否有新错误引入
5. **回滚决策** — 如发现严重劣化，可联系发布系统进行回滚

### 场景 4：接口性能问题

1. **接口分析** — 查看 P95 排行，定位最慢的接口
2. **告警定位** — 如果已触发告警，直接从告警中心进入
3. **Agent 诊断** — 查看诊断报告中的接口分析证据
4. **协作处理** — 在协作区添加评论，联系后端团队

## 附录：指标速查

### Web Vitals 阈值

| 指标 | Good | Needs Improvement | Poor |
|------|------|-------------------|------|
| LCP P75 | ≤ 2.5s | ≤ 4.0s | > 4.0s |
| INP P75 | ≤ 200ms | ≤ 500ms | > 500ms |
| CLS P75 | ≤ 0.1 | ≤ 0.25 | > 0.25 |
| TTFB P75 | ≤ 800ms | ≤ 1800ms | > 1800ms |

### 告警级别

| 级别 | 示例 | 处置 |
|------|------|------|
| S0 | 白屏率 > 5% | 立即通知值班人，5 分钟未认领升级 |
| S1 | 核心页面 LCP 劣化 > 10 分钟 | 通知团队群，建议创建治理任务 |
| S2 | 非核心页面性能劣化 | 收敛到治理看板 |
| S3 | 趋势性问题 | 进入任务池观察 |

### 术语表

| 术语 | 含义 |
|------|------|
| PV | 页面浏览量 |
| UV | 独立访问用户数（匿名去重） |
| Session | 会话，连续 30 分钟无事件结束 |
| Affected Users | 命中过性能不达标的去重用户数 |
| 归一化 | 将动态路由转化为模板（`/product/123` → `/product/:id`） |
| Agent 诊断 | AI Agent 基于只读证据生成的结构化诊断报告 |
| 治理任务 | 包含验收指标的闭环跟踪任务 |
| 观察窗口 | 修复验证的时间范围，默认 24 小时 |
